# Phase Summaries

Use this folder for Markdown stage summaries imported from related research projects.

## What Belongs Here

- Project-stage summaries that explain methods, data choices, assumptions, findings, or unresolved issues.
- Notes that may support the manuscript argument, empirical design, or modelling workflow.
- Imported Markdown files that should remain citable and traceable within this project.

## Naming

Use stable, descriptive names:

```text
YYYY-MM-DD_source-project_topic.md
```

Example:

```text
2026-05-23_westminster-fuel-risk_baseline-modelling-summary.md
```

## Provenance

At the top of each imported summary, record:

```markdown
Source project:
Original path:
Imported on:
Imported by:
Purpose in this paper:
```

Keep these summaries focused on reusable research context. Put exploratory drafts or temporary extracts in `scratch/` instead.
